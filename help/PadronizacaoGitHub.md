# Padronização do uso do GitHub

## main
Colocar **apenas** os códigos que compilam e funcionam bem!
Isso garante que sempre podemos rodar o código correto em qualquer computador.

## dev
Colocar os códigos em estado de desenvolvimento e testes.
Isso permite que analizemos os códigos uns dos outros.

## feature/nome_da_feature
Colocar os códigos referentes a uma feature específica.
Isso permite que foquemos na implementação de uma feature específica sem nos preocuparmos com influências externas.
